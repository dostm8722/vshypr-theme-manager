# CLAUDE.md — vsHypr Theme Manager

Guide for Claude Code when working on this project. Code comments are in Spanish.

---

## Overview

**vsHypr Theme Manager** is a theming system for Arch Linux + Hyprland distributed via AUR.
It applies a unified color scheme to 16 applications simultaneously from a single entry point.

| File | Purpose |
|------|---------|
| `vshypr-theme-manager.py` | Theme engine — apply, backup, inject |
| `vshypr-theme-manager` | GTK3 GUI (Python, no extension) |
| `rofi-picker.sh` | Visual Rofi picker with thumbnails + palette |
| `vshypr-theme-manager.desktop` | Desktop entry for app launchers |
| `vshypr-theme-manager.png` | App icon |

## AUR Distribution

The app is designed to be installed via AUR. All user data lives at:

```
~/.config/vshypr-theme-manager/
```

Key path constants in `vshypr-theme-manager` (GUI):

```python
_SCRIPT_DIR    = os.path.dirname(os.path.abspath(__file__))   # executable location (/usr/bin/)
_USER_DATA_DIR = os.path.expanduser("~/.config/vshypr-theme-manager")

_TC_PY      = os.path.join(_USER_DATA_DIR, "vshypr-theme-manager.py")
_TC_ROFI    = os.path.join(_USER_DATA_DIR, "rofi-picker.sh")
_TC_THEMES  = os.path.join(_USER_DATA_DIR, "themes")
_TC_CURRENT = os.path.join(_USER_DATA_DIR, "current-theme.json")
_APP_LOGO   = os.path.join(_SCRIPT_DIR,    "vshypr-theme-manager.png")  # stays with executable

_CONF_DIR   = _USER_DATA_DIR
_F_STATE    = os.path.join(_CONF_DIR, ".tcgui_state.json")
_F_CONFIG   = os.path.join(_CONF_DIR, "gui-config.json")
```

`_APP_LOGO` intentionally uses `_SCRIPT_DIR` — it lives alongside the executable, not in user data.
Everything else uses `_USER_DATA_DIR`. Never use `_SCRIPT_DIR` for data lookups.

## Theme Structure

Each theme is a directory under `themes/`:

```json
{
  "meta": {
    "name": "catppuccin",
    "display_name": "Catppuccin Mocha",
    "variant": "dark"
  },
  "colors": {
    "bg": "#hex",        "bg_alt": "#hex",
    "surface": "#hex",   "surface2": "#hex",  "surface3": "#hex",
    "overlay": "#hex",
    "fg": "#hex",        "fg_dim": "#hex",
    "accent": "#hex",    "accent_alt": "#hex", "accent_dim": "#hex",
    "red": "#hex",       "orange": "#hex",     "yellow": "#hex",
    "green": "#hex",     "teal": "#hex",       "teal_dim": "#hex",
    "blue": "#hex",      "sky": "#hex",        "mauve": "#hex",
    "pink": "#hex",      "lavender": "#hex",
    "border_active": "#hex", "border_inactive": "#hex", "shadow": "#hex"
  }
}
```

- `variant`: `"dark"` or `"light"` — controls section grouping in GUI and Rofi picker
- `display_name`: shown in GUI cards and Rofi; must not contain literal `&` (use `&amp;` — Pango markup)
- `thumb.jpg`: 80×80 px, used in Rofi picker and GUI cards
- `wallpaper.webp`: all wallpapers must be `.webp` format (converted with `cwebp -q 90`)

**Dynamic themes:** `dynamic-dark` and `dynamic-light` are generated via `matugen` from a user-selected wallpaper. They are stored like any other theme in `themes/`.

## CSS Injection System

Most config files are modified using marker-based injection:

```css
/* theme-changer: begin */
/* generated block */
/* theme-changer: end */
```

`inject_css(path, block)` in `vshypr-theme-manager.py`:
- Replaces content between markers if they exist
- Appends block at end of file if markers don't exist yet
- Content outside markers is never touched

For Hyprland/Hyprlock files (key = value format), `inject_hypr(path, block)` is used instead.

## Backup System

Two levels — both live under `~/.config/vshypr-theme-manager/backups/`:

```
backups/
├── original/                  ← written once per file, never overwritten
│   ├── kitty/theme.conf
│   ├── waybar/style.css
│   └── ...
├── 2025-06-01_14-30-00/       ← snapshot before each apply
└── 2025-06-02_09-15-22/
```

Backup structure mirrors `~/.config/` hierarchy.

## GUI — `vshypr-theme-manager`

GTK3 Python app (~2100 lines). 7 notebook tabs:

| Tab | Key class/method |
|-----|-----------------|
| Setup | `_page_setup()` — dependency + app checker, yay/paru installer |
| Config | `_page_config()` — wallpapers dir, data dir, patch rofi-picker.sh |
| Set Theme | `_page_themes()` — theme grid, `_make_theme_card()` |
| Wallpaper | `_page_wallpaper()` — file picker with 16:9 preview |
| Assets | `_page_assets()` — install bundled configs to ~/.config/ |
| vsHub | `_page_hub()` — vs ecosystem tool discovery |
| About | `_page_about()` — version, links, backups folder button |

**Important implementation details:**

- `_set_status(msg, timeout=4)` — cancels pending timeout with `GLib.source_remove()` before setting a new one; stores handle in `self._status_timeout`
- `_make_theme_card(name, display_name, is_cur, colors, theme_dir)` — takes `display_name` as parameter, does NOT re-read `colors.json`
- Theme loop builds tuples `(name, display_name, colors, theme_dir)` — reads `colors.json` once per theme
- vsHub fetch completion: `switch_to_hub=False` — must NOT switch tabs on startup
- `_page_assets()` button wiring: collect all `(btn, src_rel, dst, name, card, status_lbl)` tuples first, then wire all callbacks in a single loop to avoid double-connect

**Status bar:** `self._status_lbl` — set via `_set_status()`, auto-clears to "Ready" after timeout.

## Rofi Picker — `rofi-picker.sh`

- Uses `-format "i"` (index) to map selection → theme name via `THEME_NAMES` array
- Null bytes (`\0icon\x1f`) don't survive in bash variables — stream piped directly to rofi
- `Alt+Enter` keybinding: `-kb-custom-1 "alt+Return"`, exit code 10
- Sections: DARK / LIGHT / DYNAMIC (dynamic-dark and dynamic-light are fixed entries)
- Wallpaper picker: `~/.config/rofi/wallpaper-picker.rasi` with 4-column grid
- Wallpapers directory: `~/Pictures/wallpapers` (configurable via GUI Config tab)

## Known Workarounds

### GTK4 — catppuccin symlinks
`gtk-4.0/gtk.css` and `gtk-dark.css` may be symlinks to `/usr/share/themes/catppuccin-*/`. Detected and replaced with real files before injection.

### GTK3 — hardcoded hex in catppuccin
The catppuccin GTK3 theme uses hardcoded hex values in CSS rules, not `@define-color` references. Fix: force `gtk-theme-name=Adwaita` so user stylesheet variables take effect.

### GTK — prefer-dark color-scheme
When `gsettings color-scheme = prefer-dark`, libadwaita loads `gtk-dark.css` ON TOP of `gtk.css`. Both files must have the overrides.

### Kvantum — hardcoded GeneralColors
Kvantum ignores the Qt palette when `[GeneralColors]` is set in its kvconfig. SVGs also embed catppuccin hex values. Fix: generate a custom theme in `~/.config/Kvantum/ThemeChanger/` with a patched SVG + kvconfig with theme colors.

### dolphinrc — ColorScheme override
Dolphin and other KDE apps may have `ColorScheme=BreezeClassic` in their own rc files, taking priority over kdeglobals. Patched automatically.

### matugen 4.x — JSON structure
- Output path: `colors.<role>.default.color` (not `colors.dark.<role>`)
- Requires `--prefer saturation` for non-interactive use
- Light variant uses sub-key `"light"` instead of `"default"`

### awww — not swww
The project uses `awww`, not `swww`. Commands:
```bash
awww-daemon --daemon
awww img <path> --transition-type fade
```

### SwayNC — CSS custom properties
SwayNC uses `:root { --cc-bg: ...; }` custom properties, NOT `@define-color`.
`--noti-bg` requires `r, g, b` format (RGB components), not full hex.

### Pango markup in Rofi
`display_name` values in `colors.json` must not contain literal `&` — it breaks Pango markup and misaligns indices in the `THEME_NAMES` array. Use `&amp;` or avoid the character.

## Assets — `assets/`

Bundled config files installed to `~/.config/` via the Assets tab in the GUI:

```
assets/
├── rofi/
│   ├── config.rasi           → ~/.config/rofi/config.rasi
│   ├── spotlight.rasi        → ~/.config/rofi/spotlight.rasi
│   ├── launchpad.rasi        → ~/.config/rofi/launchpad.rasi
│   ├── config-keybinds.rasi  → ~/.config/rofi/config-keybinds.rasi
│   ├── wallpaper-grid.rasi   → ~/.config/rofi/wallpaper-grid.rasi
│   ├── theme-picker.rasi     → ~/.config/rofi/theme-picker.rasi
│   └── wallpaper-picker.rasi → ~/.config/rofi/wallpaper-picker.rasi
├── wlogout/
│   ├── layout                → ~/.config/wlogout/layout
│   └── icons/                → ~/.config/wlogout/icons/ (6 SVGs)
└── hypr/
    └── wallpaper.sh          → ~/.config/hypr/wallpaper.sh
```

`wallpaper.sh` ships with the wallpaper path set to `""` — user fills it in via the Wallpaper tab.
